this section applies to a ubuntu install and config:

to start eclipse:

!!! first set the enviroment for the crosscompiler or nothing works
when using the usb boot disk this step is not necessary and easily missed

under /opt/phytec-yogurth/ complete the path to environment-setup-cortexa9hf-neon-phytec-linux-gnueabi
then source the settings:
for example

source /opt/phytec-yogurt/BSP-Yocto-i.MX6-PD18.1.0/environment-setup-cortexa9hf-neon-phytec-linux-gnueabi

this exports all env variables necessarz for invoking the crosscompiler and linker correctlz
only then start eclipse
the sourcing step has to be repeated next time
or put it in the startup script




#/bin/bash

source /opt/phytec-yogurt/BSP-Yocto-i.MX6-PD18.1.0/environment-setup-cortexa9hf-neon-phytec-linux-gnueabi

eclipse &

Neues project in eclipse:

wie gewohnt ein C projekt erstellen mit HelloWorld template
build creates code for hostemachine, i.e. this iwll not run on mira
change project properties to:
c/c++ build -> settings / tab toolsettings (the first of many)
for compiler set command to ${CC}
for linker set command to ${CC} and in commandline pattern add ${LDFLAGS} behind $(comman)
for assembler set command to ${AS}
apply 

tab build steps (the 2nd):
post build steps 
set command to:
scp ./HelloTest root@192.168.3.10:/home/root/.

this copies the project folder over to mira


or

manually copy file to remote mira and execute:
scp /path/to/file username@a:/path/to/destination
scp HelloTest root@192.168.3.10:/home/root
ssh into mira and execute code in debug this should now work
